# 垃圾回收的类型  Garbage Colection
## Serial GC
- 最小化使用内存和并行开销

## Parallel GC
- 最大化应用程序的吞吐量


## CMS GC
- 最小化GC的中断 或 停顿时间

## Garbage Firest G1 并行回收器
- 几种精力 处理垃圾最多的区间
- 简介 后续补充：
  - 堆内存分隔为很多不想管的区间（Region）
  - 每个区间可以属于老年代或年轻代
  - 物理上不连续
- 四个阶段
  - 年轻代回收 Young Collection
  - 运行在后台的并行循环 Background Concurrent Cycle
  - 混合回收 Mixed Collection
  - 全量回收 Full GC
